# Discord 노래봇 - 개발 내역

## 프로젝트 개요

Python과 discord.py를 활용한 디스코드 음악 봇입니다. YouTube에서 음악을 검색/재생하며, 대기열 관리, 투표 스킵, 유사곡 자동재생 등의 기능을 제공합니다.

---

## 기술 스택

| 항목 | 기술 |
|------|------|
| 언어 | Python 3.12 |
| Discord API | discord.py 2.7+ (voice 확장 포함) |
| 음원 추출 | yt-dlp |
| 오디오 인코딩 | FFmpeg 8.1 |
| 패키지 관리 | uv |
| 환경 변수 | python-dotenv |

## 구현된 기능 목록

### 기본 명령어

| 명령어 | 설명 |
|--------|------|
| `/join` | 사용자가 있는 음성 채널에 봇을 입장시킵니다. |
| `/leave` | 봇을 음성 채널에서 퇴장시킵니다. |
| `/play <URL 또는 검색어>` | URL은 바로 재생, 검색어는 5개 결과를 버튼으로 제시하여 선택 후 재생합니다. |
| `/pause` | 현재 재생 중인 노래를 일시정지합니다. |
| `/resume` | 일시정지된 노래를 다시 재생합니다. |
| `/stop` | 재생을 중지하고 대기열을 초기화합니다. |

### 투표 스킵

| 명령어 | 설명 |
|--------|------|
| `/skip` | 투표를 통해 현재 노래를 건너뜁니다. |

- `/skip` 실행 시 채팅창에 **투표 안내 Embed**와 **스킵 투표 버튼**이 표시됩니다.
- 같은 음성 채널에 있는 사용자(봇 제외)의 **과반수(50% 이상)** 가 버튼을 클릭해야 스킵됩니다.
- 투표 현황(누가 투표했는지)이 Embed에 실시간으로 업데이트됩니다.
- 채널에 혼자 있는 경우 즉시 스킵됩니다.
- 60초 내에 투표가 통과되지 않으면 투표가 자동으로 만료됩니다.

### 대기열 관리

| 명령어 | 설명 |
|--------|------|
| `/queue` | 현재 대기열 목록을 Embed로 표시합니다. 최대 15곡까지 표시됩니다. |
| `/remove <번호>` | 대기열에서 해당 번호의 곡을 제거합니다. |

### 반복 및 셔플

| 명령어 | 설명 |
|--------|------|
| `/loop <모드>` | 반복 모드를 전환합니다. (끄기 / 한 곡 반복 / 전체 반복) |
| `/shuffle` | 대기열의 곡 순서를 무작위로 섞습니다. |

### 검색

| 명령어 | 설명 |
|--------|------|
| `/search <검색어>` | YouTube에서 최대 5개의 검색 결과를 보여주고, 버튼을 클릭해 선택할 수 있습니다. |

- 검색 결과가 Embed로 표시되고, 1~5 번호 버튼이 함께 나타납니다.
- 30초 내에 버튼을 클릭하면 해당 곡이 재생/대기열에 추가됩니다.
- 요청자만 버튼을 클릭할 수 있습니다.

### 현재 재생 정보

| 명령어 | 설명 |
|--------|------|
| `/nowplaying` | 현재 재생 중인 곡의 제목, URL, 길이, 요청자 정보를 Embed로 표시합니다. |

### 유사곡 자동재생

| 명령어 | 설명 |
|--------|------|
| `/autoplay` | 자동재생 기능을 켜거나 끕니다. (기본값: 켜짐) |

- 대기열이 비어있고, 반복 모드가 꺼져 있을 때 동작합니다.
- 현재 재생 중인 곡의 제목에서 키워드를 추출하여 YouTube에서 유사한 곡을 검색합니다.
- 최근 재생한 곡(최대 30곡)과 중복되지 않도록 필터링합니다.
- 상위 후보 3곡 중 하나를 무작위로 선택하여 자동 재생합니다.

### 자동 퇴장

- 음성 채널에 봇만 남으면 30초 후 자동 퇴장합니다.
- 재생이 끝나고 대기열이 비어있으면 3분(180초) 후 자동 퇴장합니다.

## 아키텍처 설계

### Cog 패턴

- `bot.py`는 봇 초기화와 Cog 로딩만 담당합니다.
- 모든 음악 관련 로직은 `cogs/music.py`의 `Music` Cog에 집중됩니다.
- 기능 확장 시 새로운 Cog 파일을 추가하면 됩니다.

### 서버별 상태 관리

- `GuildState` 데이터 클래스가 서버(guild)마다 독립적인 상태를 관리합니다.
- 대기열, 현재 곡, 반복 모드, 스킵 투표, 재생 이력 등을 포함합니다.
- 봇이 퇴장하면 해당 서버의 상태는 초기화됩니다.

### 재생 엔진

1. `extract_song()` → yt-dlp로 YouTube에서 오디오 스트림 URL 추출 (다운로드 없이 스트리밍)
2. `FFmpegOpusAudio`로 음성 채널에 스트리밍
3. 재생 완료 시 `after` 콜백 → `_advance()` 호출 → 다음 곡 결정
4. 스트림 URL 만료 시 자동 재추출 로직 포함

## 실행 방법

```bash
# 가상환경 활성화 (uv 사용)
.venv\Scripts\activate

# 봇 실행
python bot.py
```

`.env` 파일에 `BOT_TOKEN`이 설정되어 있어야 합니다.
